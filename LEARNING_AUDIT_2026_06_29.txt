Severity: CRITICAL - Directly causing poor performance

Evidence:
  BULL_TREND high_vol trades:
    Trades 1-100:   TP=20bps,  WR=35%
    Trades 101-150: TP=21bps,  WR=30%
    Trades 151-200: TP=22bps,  WR=28%
    ... (pattern continues)
    Trades 3000+:   TP=40bps,  WR=20%  ← CURRENT

  BEAR_TREND high_vol trades:
    Trades 1-100:   TP=18bps,  WR=32%
    Trades 101-150: TP=19bps,  WR=30%
    ...
    Trades 2500+:   TP=40bps,  WR=21%  ← CURRENT

Pattern: Every TP widening coincides with WR decrease

Root Cause Analysis:
  Adaptation rule: if WR < 45% → widen TP (add 1bps per 50 trades)

  Assumption: "Low WR means entries are late, need wider exit window"

  Reality:
    - Wider TP means longer time to reach exit
    - Longer time in position = more exposure to adverse moves
    - TP=10bps exits in 10 seconds, TP=40bps takes 40+ seconds
    - During 40 seconds, price can move against you more
    - Result: More SL exits, fewer TP exits

  Why WR correlated inversely with TP:
    - Entry signal quality is the REAL blocker (50-60% inherent WR ceiling)
    - TP width doesn't fix bad entry timing
    - Wider TP = delays exit of losing trades (deeper losses)
    - Tighter TP = faster exit of losing trades (smaller losses)

Impact:
  - 40bps TP creates 20% WR performance
  - Bot is caught in negative feedback loop
  - Learning system making things worse, not better

Fix Required:
  REVERSE the adaptation direction:

  Current (WRONG):
    if wr < 0.45: tp = min(tp + 0.01, 0.40)  # Widen

  Should be:
    if wr < 0.45: tp = max(tp - 0.01, 0.15)  # TIGHTEN

  Rationale: Tighten to exit losses faster, reduce damage
