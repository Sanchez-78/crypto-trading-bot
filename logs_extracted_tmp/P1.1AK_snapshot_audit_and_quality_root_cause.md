## Required Fixes

### 1. Make audit script atomic

Rewrite `scripts/p11ag_quality_audit.sh` so it reads journal once into a temp file and all counters/sample logs come from that same file.

Requirements:

- Use `mktemp`.
- Clean up with `trap`.
- Filter by:
  - current systemd PID
  - service start timestamp
  - user-provided `--since`
- Store filtered logs in one temp file, e.g.:

```bash
LOG_TMP="$(mktemp)"
trap 'rm -f "$LOG_TMP"' EXIT
journalctl ... > "$LOG_TMP"
```

Then all counts must use:

```bash
grep -c "PATTERN" "$LOG_TMP"
```

No counter may call `journalctl` directly after the snapshot is created.

Output must include:

```text
Audit snapshot lines: <n>
Audit snapshot first_ts: ...
Audit snapshot last_ts: ...
```

If the journal read fails with `Failed to iterate through journal: Bad message`, fallback to a shorter time window and print a clear warning.

### 2. Split counters by source and trade type

Current counts can mix real training trades, synthetic diagnostic trades, stale timeout-no-price, and unrelated paper exits.

Add counters:

```text
PAPER_TRAIN_ENTRY_TOTAL
PAPER_TRAIN_ENTRY_REAL
PAPER_TRAIN_ENTRY_SYNTHETIC_OR_TEST

PAPER_EXIT_TOTAL
PAPER_EXIT_TRAINING_BUCKET
PAPER_EXIT_NON_TRAINING

PAPER_TRAIN_QUALITY_EXIT_TOTAL
PAPER_TRAIN_QUALITY_EXIT_TRAINING_BUCKET
PAPER_TRAIN_QUALITY_EXIT_TIMEOUT_NO_PRICE
```

Classification rules:

- Real training entry:
  - line contains `PAPER_TRAIN_ENTRY`
  - and `bucket=C_WEAK_EV_TRAIN`
  - and source is not synthetic/test
- Training exit:
  - line contains `PAPER_EXIT`
  - and `training_bucket=C_WEAK_EV_TRAIN` or `bucket=C_WEAK_EV_TRAIN`
- Quality training exit:
  - line contains `PAPER_TRAIN_QUALITY_EXIT`
  - and `training_bucket=C_WEAK_EV_TRAIN`
- Timeout-no-price should be counted separately because it can be a quarantine path and may not correspond to a normal `PAPER_EXIT`.

Pass/fail criteria must use only training-bucket counts, not raw totals.

### 3. Add quality-exit correlation by `trade_id`

The audit should detect missing quality exits by `trade_id`, not only by counts.

Implementation can be bash/awk/python embedded in the script.

For each `PAPER_EXIT` training trade:

- Extract `trade_id` if available.
- If `PAPER_EXIT` does not include `trade_id`, classify as `exit_trade_id_missing`.
- Search for matching `PAPER_TRAIN_QUALITY_EXIT trade_id=<same>`.
- Report missing list:

```text
QUALITY_EXIT_MISSING_BY_TRADE_ID: 0
Missing quality exits:
- trade_id=... symbol=... exit_time=...
```

If `PAPER_EXIT` currently does not log `trade_id`, add it in `paper_trade_executor.py` for paper training closes.

### 4. Add explicit cost-edge bypass context

Observed:

```text
[PAPER_TRAIN_QUALITY_ENTRY] ... cost_edge_ok=False ... source=training_sampler ...
```

This is acceptable only if it was intentionally admitted by the P1.1AE bootstrap bypass. The log currently does not prove why it was accepted.

Add fields to `PAPER_TRAIN_ENTRY` and `PAPER_TRAIN_QUALITY_ENTRY`:

```text
cost_edge_ok=<bool>
cost_edge_bypassed=<bool>
cost_edge_bypass_reason=<none|paper_train_bootstrap_strict_take|portfolio_gate_training_bootstrap|...>
bootstrap_closed_trades=<int>
mode=<paper_train|live|real|...>
```

Rules:

- If `cost_edge_ok=False` and accepted:
  - `cost_edge_bypassed=True` must be present.
  - bypass reason must be non-empty.
- If live/real:
  - bypass must never occur.
- Add anomaly:

```text
[PAPER_TRAIN_ANOMALY] type=cost_edge_false_without_bypass ...
```

### 5. Add economic root-cause summary

Add a compact 5-minute rolling summary focused on why LOSS/FLAT happens:

```text
[PAPER_TRAIN_ECON_SUMMARY]
window_s=300
closed=<n>
win=<n>
loss=<n>
flat=<n>
avg_pnl=<x>

avg_sl_pct=<x>

sl_touch_rate=<x>

near_tp_timeout=<n>
near_sl_timeout=<n>
avg_exit_efficiency=<x>
score_missing=<n>
expected_move_corrected=<n>
cost_edge_bypassed=<n>
by_side=[BUY:n=.. wr=.. avg_pnl=..,SELL:n=.. wr=.. avg_pnl=..]

```

Definitions:

- `near_tp_timeout`: TIMEOUT where MFE reached at least 70% of TP distance but TP did not close.
- `near_sl_timeout`: TIMEOUT where MAE reached at least 70% of SL distance.
- `both_touch_rate`: both TP and SL touched during hold, suggesting TP/SL too tight or volatile/noisy sample.
- `exit_efficiency` should be side-aware and bounded where possible.
