# P1.1Z-hotfix — Fix Missing Safe Helpers in Paper State Load/Reconcile

## Goal
Fix production startup failure:

```text
[PAPER_STATE_LOAD_ERROR] source=data/paper_open_positions.json err=name '_safe_float' is not defined
```

This prevents stale position normalization/reconcile, so old `C_WEAK_EV_TRAIN` positions with `timeout_s=900` keep blocking new training entries.

## Root Cause
`P1.1Z` introduced startup normalization/reconcile code in `paper_trade_executor.py` that calls `_safe_float`, but `_safe_float` is not defined in that module/scope.

## Required Patch
In `src/services/paper_trade_executor.py`, add local safe helpers near the top of the file, before any startup loader/reconcile function can call them:

```python
def _safe_float(value, default=0.0):
    try:
        if value is None:
            return float(default)
        return float(value)
    except Exception:
        return float(default)


def _safe_int(value, default=0):
    try:
        if value is None:
            return int(default)
        return int(float(value))
    except Exception:
        return int(default)
```

Keep these helpers local in `paper_trade_executor.py`. Do not import them from another module.
