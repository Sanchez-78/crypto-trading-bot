# V10.13u+13 — Close Loop Final Recovery

## Problem
V10.13u+12 is live and TTL works, but close still loops forever.

Logs:
- `CLOSE_SKIP_DUPLICATE ... attempts=1000+`
- `CLOSE_LOCK_STALE_RELEASE ... stale_count=13`
- `POSITION_CLOSE_STUCK ... action=reconcile_required`
- same key reacquires every 20s

Meaning: lock recovery works, but position is never removed/terminal after failed close.

## Goal
Stop infinite close loops. After repeated stale close locks, force terminal reconciliation.

Do NOT change:
- PF formula
- EV-only logic
- Firebase read/write behavior
- canonical PnL formula
- TP/SL strategy

## Files
- `src/services/trade_executor.py`
- `src/services/exit_attribution.py`
- `tests/test_v10_13u_patches.py`

### 2) Add force reconcile helper

```python
def _force_reconcile_stuck_close(key: str, meta: dict | None = None, reason: str = "stale_close_loop") -> bool:
    """
    Last-resort recovery for positions that stay already_closing forever.
    Must remove/terminalize the open position and block immediate reacquire.
    """
    import time
    meta = meta or {}
    symbol = meta.get("symbol") or key.split(":")[0]
    changed = False

    try:
        # Adapt to existing open position container.
        # If positions are dict-like by symbol:
        if isinstance(open_positions, dict):
            if symbol in open_positions:
                open_positions.pop(symbol, None)
                changed = True
            else:
                # fallback: remove matching key/symbol inside values if needed
                for k, p in list(open_positions.items()):
                    if isinstance(p, dict) and p.get("symbol", k) == symbol:
                        open_positions.pop(k, None)
                        changed = True

        # If positions are list-like:
        elif isinstance(open_positions, list):
            before = len(open_positions)
            open_positions[:] = [
                p for p in open_positions
                if not (isinstance(p, dict) and p.get("symbol") == symbol)
            ]
            changed = len(open_positions) != before
    except Exception as e:
        log.exception(f"[CLOSE_FORCE_RECONCILE_FAIL] key={key} symbol={symbol} err={e}")

    _CLOSING_POSITIONS.pop(key, None)
    _RECENTLY_CLOSED[key] = {
        "ts": time.time(),
        "symbol": symbol,
        "status": "force_reconciled",
        "reason": reason,
    }

    log.error(
        f"[CLOSE_FORCE_RECONCILE] key={key} symbol={symbol} "
        f"reason={reason} stale_count={_STALE_CLOSE_COUNTS.get(key, 0)} "
        f"attempts={meta.get('attempts', 0)} changed={changed}"
    )
    return changed
```

If `open_positions` has a different name, use the actual existing container. Do not create a new one.

### 3) Trigger force reconcile from stale release

In `_release_stale_close_lock(key, ...)`, after incrementing stale count:

```python
stale_count = _STALE_CLOSE_COUNTS.get(key, 0)
attempts = int((meta or {}).get("attempts", 0))

if stale_count >= CLOSE_LOCK_FORCE_RECONCILE_AFTER or attempts >= CLOSE_LOCK_MAX_ATTEMPTS:
    _force_reconcile_stuck_close(key, meta, reason="stale_lock_threshold")
    return
```

Important: after force reconcile, do not immediately reacquire the same key.

### 4) Cleanup before duplicate/acquire

At start of `_try_acquire_close_lock(...)`:

```python
_cleanup_close_locks()
```

Then if key in `_RECENTLY_CLOSED` and TTL not expired, return not acquired.

Duplicate log only every 10s/key:

if now - meta.get("last_log", 0) >= CLOSE_DUP_LOG_INTERVAL_S:

No dozens-per-second spam.
