# P1.1V — Fix paper_train learning loop (token-safe)

## Context
Production traceback:
```txt
[LEARNING_UPDATE_ERROR] err='int' object is not iterable fn=update_from_paper_trade
paper_trade_executor.py line 691: record_training_closed(bucket=canon["bucket"], outcome=canon["outcome"])
paper_training_sampler.py line 619: _training_metrics["closed_1h"] += 1
TypeError: 'int' object is not iterable
```

Smoke replay proves `update_from_paper_trade(trade)` returns `True`. Failure is telemetry after learning.

## Root cause
`_training_metrics["closed_1h"]` is list/deque-like, but code does `+= 1`. For list/deque, `+=` expects iterable, so int causes error. This telemetry exception is incorrectly logged as learning failure.

## Hard rules
- Do not change strategy, thresholds, buckets, EV logic, or live-order safety.
- Keep `TRADING_MODE=paper_train`, `ENABLE_REAL_ORDERS=false`.
- Fix only telemetry, metrics, duplicate update paths, and confirmed `is_paper_mode` bug.
- No Firebase write amplification.

## Tasks

### 1) Fix `record_training_closed()` counters
File: `src/services/paper_training_sampler.py`

Add safe helpers:
```python
def _metric_add_event(key: str, ts: float | None = None) -> None:
    import time
    now = float(ts or time.time())
    v = _training_metrics.get(key)
    if hasattr(v, "append"):
        v.append(now)
    elif isinstance(v, (int, float)):
        _training_metrics[key] = int(v) + 1
    elif v is None:
        _training_metrics[key] = [now]
    else:
        _training_metrics[key] = [now]

def _metric_inc_counter(key: str, n: int = 1) -> None:
    import time
    v = _training_metrics.get(key, 0)
    if isinstance(v, (int, float)):
        _training_metrics[key] = int(v) + n
    elif hasattr(v, "append"):
        v.append(time.time())
    else:
        _training_metrics[key] = n
```

Replace unsafe increments:
```python
_training_metrics["closed_1h"] += 1
```
with:
```python
_metric_add_event("closed_1h")
```

Search and replace every `_training_metrics[...] += 1` safely. `record_training_closed()` and `record_training_learning_update()` must never raise. Telemetry errors log as `[PAPER_TRAIN_METRICS_ERROR]`, not `[LEARNING_UPDATE_ERROR]`.

### 2) Decouple learning from telemetry
File: `src/services/paper_trade_executor.py`

`_safe_learning_update_for_paper_trade()` must:
1. canonicalize trade
2. call `update_from_paper_trade(canon)`
3. log learning result
4. call telemetry in separate try/except

Target:
```python
ok = False
try:
    ok = bool(update_from_paper_trade(canon))
    log.info(
        "[LEARNING_UPDATE] ok=%s source=paper_train symbol=%s bucket=%s outcome=%s net_pnl_pct=%.4f",
        ok, canon["symbol"], canon["bucket"], canon["outcome"], canon["net_pnl_pct"],
    )
except Exception as e:
    log.exception(
        "[LEARNING_UPDATE_ERROR] err=%r symbol=%s bucket=%s fn=update_from_paper_trade",
        e, canon.get("symbol"), canon.get("bucket"),
    )
    return False

try:
    record_training_closed(bucket=canon["bucket"], outcome=canon["outcome"])
    if ok:
        record_training_learning_update()
except Exception as e:
    log.warning(
        "[PAPER_TRAIN_METRICS_ERROR] err=%r symbol=%s bucket=%s",
        e, canon.get("symbol"), canon.get("bucket"),
    )

return ok
```
