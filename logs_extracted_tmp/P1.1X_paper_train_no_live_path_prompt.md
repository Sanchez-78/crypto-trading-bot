# P1.1X — Hard Paper-Mode Routing: Stop LIVE_ORDER_DISABLED Spam

## Goal
Fix routing bug: `paper_train` is now opening/closing paper trades and learning correctly, but accepted signals still reach live-order safety guard and spam `[LIVE_ORDER_DISABLED]`.

Current production evidence:
- `PAPER_TRAIN_ENTRY=3 / 15m`
- `PAPER_EXIT=3 / 15m`
- `LEARNING_UPDATE ok=True=3 / 15m`
- error count = `0`
- but `LIVE_ORDER_DISABLED=71 / 15m`

This means learning loop works. Do **not** change learning logic. Only fix routing/logging so paper modes never enter live order path.

## Hard Rules
- No live orders unless all are true:
  - `TRADING_MODE=live_real`
  - `ENABLE_REAL_ORDERS=true`
  - `LIVE_TRADING_CONFIRMED=true`
- In `paper_train` and `paper_live`, code must return before any live-order execution/safety guard.
- Do not touch EV logic, sampler thresholds, bucket metrics, learning update, TP/SL, or risk math.
- Preserve `P1.1V` learning success path.
- No Firebase write amplification.

## Root Cause Hypothesis
`live_trading_allowed()` / live execution guard is still called after RDE `ACCEPT/TAKE` in paper modes. The guard correctly blocks real orders, but it should not be reached in `paper_train`.

## Required Fix
### 1) Audit `src/services/trade_executor.py`
Find all paths that log or call:
- `[LIVE_ORDER_DISABLED]`
- `live_trading_allowed()`
- live order placement / exchange order submission
- `_execute_live*`, `place_order`, `create_order`, `open_position`, or equivalent

### 2) Add a hard paper-mode return before live path
In `handle_signal()` or the final execution branch, after RDE decision is accepted and after required paper position fields are available, add:

```python
mode = get_trading_mode()
if mode in ("paper_train", "paper_live"):
    # open paper position / training position, then return
    # must never call live_trading_allowed() or live order code
    return
```

For `paper_train` accepted `TAKE` signals:
- route to paper position with bucket `A_STRICT_TAKE`
- include metadata: `paper_source`, `training_bucket`, `original_decision`, `rde_decision`, `cost_edge_ok`, `expected_move_pct`, `required_move_pct`, `side_inferred`, `score_at_entry`, `features`, `regime`, `max_hold_s`
- then return immediately

For rejected/blocked signals:
- keep existing sampler route (`C_WEAK_EV_TRAIN`, etc.)
- then return immediately

For `paper_live`:
- open normal paper position or return according to existing behavior
- never call live guard

### 3) Restrict `[LIVE_ORDER_DISABLED]` logging
`[LIVE_ORDER_DISABLED]` should only be emitted when `TRADING_MODE=live_real` but one of the safety flags is false. In paper modes, replace it with either no log or a very rare debug/summary log.

Acceptable behavior:
```python
if mode != "live_real":
    return False  # silent; paper routing should have already returned
```

### 4) Add regression tests
Add tests that would fail on current behavior:

1. `paper_train` + accepted TAKE:
   - opens/routes paper position
   - returns before live order code
   - does not call `live_trading_allowed()`
   - emits no `[LIVE_ORDER_DISABLED]`

2. `paper_train` + duplicate/rejected signal:
   - routes to sampler
   - no live guard call

3. `paper_live` accepted signal:
   - no live guard call
   - no `[LIVE_ORDER_DISABLED]`

4. `live_real` with missing flags:
   - live guard blocks
   - `[LIVE_ORDER_DISABLED]` may log, throttled

5. `live_real` with all flags true:
   - live guard allows

Use monkeypatch/mocks so any live order function raises if called in paper modes.
