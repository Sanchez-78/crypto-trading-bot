## Diagnosis
P1.1AH is successful as a diagnostic layer: `PAPER_TRAIN_QUALITY_ENTRY == PAPER_TRAIN_ENTRY` and `PAPER_TRAIN_QUALITY_MISMATCH == 0`.

But production shows these actual defects:

1. **SELL TP/SL are inverted in paper training.**
   - Current SELL example: `tp > entry` and `sl < entry`.
   - Correct SELL semantics: `tp < entry`, `sl > entry`.
   - Current log also reports `tp_pct=-1.200`, `sl_pct=-1.200`, `rr=0.000`, proving distance normalization is wrong.

2. **`expected_move_pct` is likely an absolute ATR/price-unit value mislabeled as percent.**
   - BTC: `expected_move_pct=11.816` and `atr=11.81575234` are nearly identical.
   - For BTC at `80512`, ATR `11.816` price units is about `0.0147%`, not `11.816%`.

3. **Canonical score is not propagated into quality logs.**
   - Accepted entries show `score_raw=0.000 score_final=0.000`, triggering `score_zero_but_take`.
   - Earlier RDE logs showed `decision=TAKE`, so either score fields are missing from the routed candidate or defaulting to zero.

4. **Exit quality logging is not guaranteed.**
   - Audit: `PAPER_EXIT=1`, `PAPER_TRAIN_QUALITY_EXIT=0`.
   - Every training `PAPER_EXIT` must emit exactly one quality exit log before the position is removed.

5. **Learning update logging/audit is inconsistent.**
   - Audit: `LEARNING_UPDATE ok=True=0`, but `Latest Total trades in LM=1`.
   - Either the update happened without the expected log pattern or the audit script misses the current success log format.

## Goal
Implement a minimal, safe, regression-tested P1.1AI patch. Do **not** change live/real order execution behavior. Scope changes to paper training / diagnostics unless a shared helper is purely read-only or test-covered.

## Required Fixes

### 1. Side-aware paper-training TP/SL normalization
Inspect `src/services/paper_trade_executor.py`, `src/services/trade_executor.py`, and any helper that builds paper training positions.

Create or repair one canonical helper, for example:

```python
def normalize_paper_tp_sl(side: str, entry: float, tp: float, sl: float) -> dict:
    """Return side-valid TP/SL distances and RR for paper training.

    BUY:  tp > entry, sl < entry
    SELL: tp < entry, sl > entry
    Distances must be positive percentages.
    """
```

Rules:

```text
BUY:
  tp_valid = tp > entry
  sl_valid = sl < entry
  tp_pct = abs(tp - entry) / entry * 100
  sl_pct = abs(entry - sl) / entry * 100

SELL:
  tp_valid = tp < entry
  sl_valid = sl > entry
  tp_pct = abs(entry - tp) / entry * 100
  sl_pct = abs(sl - entry) / entry * 100

rr = tp_pct / sl_pct if sl_pct > 0 else 0
```

If a SELL position arrives with BUY-style levels (`tp > entry`, `sl < entry`), repair it only in paper training by swapping/reflecting distances around entry:

```text
bad SELL input: tp_above, sl_below
sell_tp = entry - abs(tp_above - entry)
sell_sl = entry + abs(entry - sl_below)
```

Log once when repaired:

```text
[PAPER_TRAIN_TP_SL_REPAIRED] trade_id=... symbol=... side=SELL old_tp=... old_sl=... new_tp=... new_sl=... reason=sell_levels_inverted
```

Fail closed only if levels are impossible after repair. Do not allow paper training positions with invalid TP/SL to be opened silently.

### 2. Fix `expected_move_pct` units
Trace where `expected_move_pct` is assigned.

Do not treat ATR absolute price units as percent. If the source is ATR absolute, log it as `atr_abs` and compute:

```python
expected_move_pct = abs(expected_move_abs) / entry * 100.0
```

If the source is already percent, preserve it. Avoid heuristic-only conversion if a named source field exists. Prefer explicit source fields:

```text
expected_move_abs
expected_move_pct
atr_abs
atr_pct
```

Quality log should include enough fields to disambiguate:

```text
expected_move_pct=... expected_move_src=atr_abs|candidate_pct|unknown atr_abs=... atr_pct=...
```

Pass criteria:

```text
BTC ATR around 11.8 at entry 80512 must not log expected_move_pct=11.8.
It should be around 0.0147 unless a deliberate multiplier is applied and named.
```

### 3. Propagate canonical score fields into paper quality logs
Trace the accepted/routed candidate from RDE → portfolio gate → training sampler → paper position.

Quality entry must not default missing scores to zero without marking that they were missing.

Implement robust extraction order, for example:

```text
score_raw candidates: candidate.score_raw, candidate.raw_score, decision.score_raw, canonical_decision.score_raw
score_final candidates: candidate.score_final, candidate.score, decision.score_final, canonical_decision.score_final
```

If absent:

```text
score_raw=None score_final=None score_missing=True
[PAPER_TRAIN_ANOMALY] type=score_missing_for_take symbol=... source=...
```

Only emit `score_zero_but_take` when the score is truly present and equal to zero.

### 4. Guarantee quality exit log for every training paper exit
Find all paper-close paths: timeout, TP, SL, trailing, manual/debug close if present.

For every position where `training_bucket == C_WEAK_EV_TRAIN` or `bucket == C_WEAK_EV_TRAIN`, emit exactly one:

```text
[PAPER_TRAIN_QUALITY_EXIT] trade_id=... symbol=... side=... outcome=... mfe_pct=... mae_pct=... exit_efficiency=... hold_s=... reason=...
```

The quality exit must happen before the position is removed from memory/disk.

If any required field is missing, still emit the quality exit with safe defaults and also emit:

```text
[PAPER_TRAIN_ANOMALY] type=quality_exit_missing_field field=... trade_id=...
```

Do not let missing diagnostics block learning updates.

### 5. Fix or align learning update success logs/audit
Production shows `Total trades in LM` incrementing while `LEARNING_UPDATE ok=True` is not counted.

Make the success path explicit and stable:

```text
[LEARNING_UPDATE] ok=True source=paper_closed_trade symbol=... regime=... bucket=... outcome=... net_pnl_pct=...
[LM_STATE_AFTER_UPDATE] source=paper_closed_trade symbol=... regime=... before_total=... after_total=... key_count=...
```

Update `scripts/p11ag_quality_audit.sh` so it counts both:

```text
LEARNING_UPDATE ok=True
LM_STATE_AFTER_UPDATE
Latest Total trades in LM
```

If `PAPER_EXIT > 0` and `Latest Total trades in LM` increased, the audit should not report a false critical failure just because one log pattern changed.
