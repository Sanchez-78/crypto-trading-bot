The prior fix still fails, probably because `update_from_paper_trade()` still calls old internals that iterate scalar feature values.

Implement `update_from_paper_trade(trade: dict) -> bool` as a self-contained safe path:

Required behavior:
- never calls `record_features()` unless `record_features()` is made scalar-safe first
- never calls old `lm_update()` unless proven scalar-safe
- never iterates over scalar feature values
- accepts `features` dict containing int/float/bool/str/None
- only iterates `features.items()`
- skips or stores scalar values directly; does not do `for x in value`

Safe feature handling:
```python
def _safe_features(raw):
    if not isinstance(raw, dict):
        return {}
    out = {}
    for k, v in raw.items():
        if isinstance(k, str):
            if isinstance(v, (int, float, bool, str)) or v is None:
                out[k] = v
            elif isinstance(v, (list, tuple)):
                out[k] = list(v)
            elif isinstance(v, dict):
                out[k] = dict(v)
            else:
                out[k] = str(v)
    return out
```
