## Patch A — canonical economic snapshot
In `src/services/canonical_metrics.py`, add or verify a single function:

```python
def canonical_economic_snapshot(closed_trades: list[dict]) -> dict:
    """Single source of truth for dashboard + LM economic health.

    Must include flats/scratch in counts and exit percentages, but PF/WR may use decisive
    wins/losses only. Never silently ignore SCRATCH_EXIT/STAGNATION_EXIT for scratch%.
    """
```

Return at minimum:

```python
{
  "total": int,                 # all closed canonical trades
  "wins": int,                  # net_pnl > epsilon
  "losses": int,                # net_pnl < -epsilon
  "flats": int,                 # otherwise
  "decisive": int,              # wins+losses
  "wr": float | None,           # wins/(wins+losses)
  "profit_factor": float,       # gross_wins/abs(gross_losses), canonical dashboard formula
  "expectancy": float,          # mean net_pnl across canonical closed trades
  "net_pnl": float,
  "scratch_count": int,         # exit reason contains SCRATCH or flat/scratch classified
  "scratch_rate": float,        # scratch_count/total
  "timeout_count": int,
  "timeout_rate": float,
  "stagnation_count": int,
  "stagnation_rate": float,
  "exit_breakdown": dict,
  "source": "canonical_economic_snapshot",
}
```

Normalization rules:

- Read PnL from first available key:
  - `net_pnl`, `pnl_net`, `pnl`, `profit`, `realized_pnl`, `result_pnl`
- Read symbol from `symbol` or `sym`.
- Read regime from `regime` or `reg`.
- Read exit reason from `exit_reason`, `reason`, `close_reason`, `exit_type`, `type`.
- Treat `SCRATCH_EXIT`, `STAGNATION_EXIT`, `TIMEOUT_FLAT`, `replaced` as closed trades, not missing data.
- PF must match dashboard `Profit Factor`. If dashboard uses `canonical_profit_factor()`, call that directly.
