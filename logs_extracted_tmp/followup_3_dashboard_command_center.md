# Follow-up 3 — PřehledScreen jako command center

Řeš jen první záložku: Přehled.

## Cíl
Musí to být command center, ne sterilní landing page.

## Přehled musí obsahovat
- stav bota
- freshness / stale info
- equity
- zůstatek
- denní PnL
- nerealizovaný PnL
- otevřené pozice
- quick KPI: úspěšnost, počet obchodů, profit factor
- quick risk / exposure
- top pár
- nejhorší pár
- dominantní rejection reason
- learning health
- poslední 3–5 obchodů
- warningy
- 1 hlavní mini graf
- 1 interpretive summary

## Pravidla
- vše česky
- každá důležitá položka má stručný kontext
- žádný chaos
- žádné NaN fallbacky
- respektovat původní design DNA

## Povinný výstup
1. Struktura Přehledu
2. Použité komponenty
3. Změněné soubory
4. Konkrétní Kotlin/Compose kód
5. Validace

Neřeš ostatní záložky.
