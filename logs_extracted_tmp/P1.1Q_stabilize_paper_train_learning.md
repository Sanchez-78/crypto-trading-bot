# CryptoMaster P1.1Q — Stabilize Paper Train Learning + Stop Patch Drift

## Goal
Stop adding features. Fix the current end-to-end `paper_train` loop so the bot:
1. opens paper training trades,
2. closes them,
3. persists state,
4. updates learning without errors,
5. updates bucket metrics without errors,
6. emits bounded logs only.

Current production evidence:
- `TRADING_MODE=paper_train`
- `PAPER_TRAIN_ENTRY` appears
- `PAPER_EXIT` appears
- `LEARNING_UPDATE` appears
- but real learning path still throws:
  - `[LEARNING_UPDATE_ERROR] err='int' object is not iterable symbol=... bucket=C_WEAK_EV_TRAIN`
  - `[BUCKET_METRICS_ERROR] argument of type 'NoneType' is not iterable`
- skip spam is reduced but still noisy.

## Hard Rules
- Do not add new trading logic.
- Do not add new buckets.
- Do not tune strategy thresholds except logging throttle/caps if necessary.
- No live orders. Keep `ENABLE_REAL_ORDERS=false`.
- Fix by reading actual function signatures, not guessing.
- Preserve current paper_train behavior: entries/exits must continue.
- All updates must be backward compatible with legacy open paper positions.
- No Firebase write amplification.

## Phase 2 — Single canonical closed-trade adapter
In `paper_trade_executor.py`, create one canonical sanitizer used for every closed paper trade before learning/metrics:

```python
def _canonical_closed_paper_trade(raw: dict) -> dict:
    # return a new dict, never mutate raw
    # guarantee:
    # symbol: str
    # regime: str
    # side: "BUY"|"SELL"|"UNKNOWN"
    # bucket: str, prefer training_bucket, then explore_bucket, else "UNKNOWN"
    # training_bucket/explore_bucket both filled consistently
    # sub_bucket: str|None, never iterable-required
    # tags: list[str]
    # features: dict
    # score/ws: float
    # net_pnl_pct: float
    # pnl_decimal: net_pnl_pct / 100.0
    # outcome: WIN|LOSS|FLAT|UNKNOWN
    # exit_reason: str
```

Rules:
- `bucket = trade.get("training_bucket") or trade.get("explore_bucket") or "UNKNOWN"`
- set both `training_bucket` and `explore_bucket` to bucket when missing.
- `features` must always be a dict. If missing/non-dict, use `{}`.
- `tags`: `None -> []`, `str -> [str]`, list/tuple/set -> list of strings, else `[]`.
- numeric conversion must be exception-safe.

## Phase 5 — Deduplicate close/update calls

## Phase 6 — Reduce skip log noise without hiding summaries

- per-event skip logs should be DEBUG or throttled to 60s per `(reason, symbol, side, bucket)`.
- INFO should emit periodic summary only:

- Keep `PAPER_TRAIN_ENTRY`, `PAPER_EXIT`, `LEARNING_UPDATE`, `PAPER_TRAIN_HEALTH` at INFO.
