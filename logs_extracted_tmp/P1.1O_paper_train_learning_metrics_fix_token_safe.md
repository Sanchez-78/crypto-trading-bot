# P1.1O — Fix paper_train learning + bucket metrics type safety

## Context
Prod commit before fix: `48c3579 P1.1N`.
Mode:
```env
TRADING_MODE=paper_train
ENABLE_REAL_ORDERS=false
LIVE_TRADING_CONFIRMED=false
PAPER_TRAINING_ENABLED=true
```

Prod errors:
```text
[LEARNING_UPDATE_ERROR] Failed to update learning on training trade: 'int' object is not iterable
[PAPER_STATE_SAVE_ERROR] err=[Errno 13] Permission denied: 'data/paper_open_positions.json'
[BUCKET_METRICS_ERROR] argument of type 'NoneType' is not iterable
```

Goal: stabilize paper_train loop:
signal/reject/block -> PAPER_TRAIN_ENTRY -> PAPER_EXIT -> LEARNING_UPDATE -> bucket metrics.
No live orders. No tick-level Firebase writes.

## Hard rules
- Do not enable real orders.
- Do not change `live_real` behavior.
- Do not loosen live gates.
- Keep paper/live metrics separated.
- All learning/metrics/state calls must be exception-safe.

## Task 0 — Deployment permission note
Do not chmod/chown from Python.
But add better logs for `PermissionError`.
Server operator will fix owner of `/opt/cryptomaster/data`.
