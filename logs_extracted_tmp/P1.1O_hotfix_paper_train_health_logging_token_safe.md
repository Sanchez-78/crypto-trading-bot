# P1.1O-hotfix — Fix PAPER_TRAIN_HEALTH logging TypeError

## Context
Production commit: `e144d71`  
Mode:
```env
TRADING_MODE=paper_train
ENABLE_REAL_ORDERS=false
LIVE_TRADING_CONFIRMED=false
PAPER_TRAINING_ENABLED=true
```

Prod traceback:
```text
--- Logging error ---
TypeError: %d format: a real number is required, not list
File: src/services/paper_training_sampler.py
Function: _maybe_log_training_health()
Message: [PAPER_TRAIN_HEALTH] open=... closed_1h=%d entries_1h=%d target_1h=%d learning_updates_1h=%d status=%s
Arguments: ([], 1, 6, 0, 'STARVED')
```

## Root cause
`_maybe_log_training_health()` passes `open_positions` list into `%d`. It must log `len(open_positions)`.

## Goal
Fix logging only. No strategy changes. No live trading changes. No new features.

## Hard rules
- Do not enable real orders.
- Do not change live_real behavior.
- Do not start P1.2.
- Do not add new training logic.
- Patch only the health log type bug.

## Tasks

### 1. Patch `src/services/paper_training_sampler.py`
Find `_maybe_log_training_health()` and the `[PAPER_TRAIN_HEALTH]` log.

Add a safe counter helper near the function or inline:

```python
def _safe_count_open_positions(open_positions) -> int:
    if isinstance(open_positions, dict):
        return len(open_positions)
    if isinstance(open_positions, list):
        return len(open_positions)
    if open_positions is None:
        return 0
    try:
        return int(open_positions)
    except Exception:
        return 0
```

Before logging:

```python
open_count = _safe_count_open_positions(open_positions)
```

Change log call to pass `open_count`, not `open_positions`:

```python
log.info(
    "[PAPER_TRAIN_HEALTH] open=%d closed_1h=%d entries_1h=%d target_1h=%d learning_updates_1h=%d status=%s",
    open_count,
    closed_1h,
    entries_1h,
    target_1h,
    learning_updates_1h,
    status,
)
```

### 2. Tests
Add or update tests:
