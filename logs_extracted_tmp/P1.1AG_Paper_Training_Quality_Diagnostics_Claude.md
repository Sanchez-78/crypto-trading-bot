This is now a **training quality / entry quality / exit geometry** problem, not a pipeline-connectivity problem.

## Objective

Implement **P1.1AG: Paper Training Quality Diagnostics**.

The patch must answer:

1. Are training entries losing because direction is wrong?
2. Are regimes wrong at entry or changing before exit?
3. Is 300s max hold too long / too short?
4. Are TP/SL distances unrealistic versus actual move?
5. Are weak routed samples from `PORTFOLIO_GATE:*` worse than `STRICT_TAKE_ROUTED_TO_TRAINING`?
6. Is `cost_edge` bypass admitting too many low-quality samples?
7. Is expected_move_pct inflated or malformed?
8. Are paper_train entries entering too frequently into the same micro-market condition?

## Required implementation

### 1. Add entry snapshot diagnostics

When a paper training position opens, log one structured line:

```text
[PAPER_TRAIN_QUALITY_ENTRY] trade_id=<id> symbol=<sym> side=<BUY|SELL> source=<source> bucket=<bucket> training_bucket=<bucket> regime=<regime> ev=<float> p=<float> score_raw=<float> score_final=<float> coh=<float> expected_move_pct=<float> cost_edge_ok=<bool> entry=<price> tp=<price> sl=<price> tp_pct=<float> sl_pct=<float> rr=<float> atr=<float> spread=<float> hold_limit_s=<int>
```

Use best-effort field extraction. If a field is unavailable, log `na`, not an exception.

Important:
- `score_raw` / `score_final` must use canonical decision fields, not stale `score=0.000`.
- Keep `[SIGNAL_RAW]` unchanged unless the existing bug is obvious and trivial to fix safely.

---

### 2. Add exit quality diagnostics

When a paper training position closes, log:

```text
[PAPER_TRAIN_QUALITY_EXIT] trade_id=<id> symbol=<sym> side=<BUY|SELL> source=<source> entry_regime=<regime> exit_regime=<regime> training_bucket=<bucket> reason=<TP|SL|TIMEOUT|...> outcome=<WIN|LOSS|FLAT> entry=<price> exit=<price> net_pnl_pct=<float> mfe_pct=<float> mae_pct=<float> hold_s=<int> hold_limit_s=<int> touched_tp=<bool> touched_sl=<bool> exit_efficiency=<float>
```

Definitions:
- `mfe_pct`: max favorable excursion during the trade.
- `mae_pct`: max adverse excursion during the trade.
- For BUY:
  - favorable = `(max_seen - entry) / entry * 100`
  - adverse = `(min_seen - entry) / entry * 100`
- For SELL:
  - favorable = `(entry - min_seen) / entry * 100`
  - adverse = `(entry - max_seen) / entry * 100`
- `exit_efficiency`:
  - if `mfe_pct > 0`: `net_pnl_pct / mfe_pct`
  - else `0.0`
- Track `max_seen` and `min_seen` inside paper open position state.

Clamp/log defensively. Never crash on missing fields.

---

### 3. Add rolling summary every 5 minutes

Add a low-volume summary:

```text
[PAPER_TRAIN_QUALITY_SUMMARY] window_s=300 closed=<n> win=<n> loss=<n> flat=<n> wr=<float> avg_pnl=<float> avg_mfe=<float> avg_mae=<float> timeout_rate=<float> by_source={<source>: {n, wr, avg_pnl}} by_regime={<regime>: {n, wr, avg_pnl}}
```

Keep it compact. Avoid huge JSON blobs. Limit source/regime maps to top entries if necessary.

---

### 4. Add anomaly logs

Log these only when triggered:

```text
[PAPER_TRAIN_ANOMALY] type=score_zero_but_take symbol=<sym> ev=<float> score_raw=<float> score_final=<float> source=<source>
[PAPER_TRAIN_ANOMALY] type=expected_move_extreme symbol=<sym> expected_move_pct=<float> source=<source>
[PAPER_TRAIN_ANOMALY] type=tp_sl_invalid symbol=<sym> entry=<price> tp=<price> sl=<price> side=<side>
[PAPER_TRAIN_ANOMALY] type=regime_flip symbol=<sym> entry_regime=<r1> exit_regime=<r2> outcome=<outcome>
[PAPER_TRAIN_ANOMALY] type=timeout_near_tp symbol=<sym> mfe_pct=<float> tp_pct=<float> net_pnl_pct=<float>
[PAPER_TRAIN_ANOMALY] type=timeout_deep_adverse symbol=<sym> mae_pct=<float> sl_pct=<float> net_pnl_pct=<float>
```

Suggested thresholds:
- `expected_move_extreme`: expected_move_pct > 5.0 unless already justified by code comments.
- `timeout_near_tp`: TIMEOUT and `mfe_pct >= 0.75 * tp_pct`.
- `timeout_deep_adverse`: TIMEOUT and `abs(mae_pct) >= 0.75 * abs(sl_pct)`.
