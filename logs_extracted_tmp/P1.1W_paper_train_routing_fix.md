# P1.1W — Fix paper_train accepted-signal routing, stop LIVE_ORDER_DISABLED spam

## Goal
In `TRADING_MODE=paper_train`, the bot must never reach live order placement code. Accepted TAKE signals must be handled as paper-training/paper-executor trades and then return. Keep real orders disabled.

## Current production status
P1.1V fixed learning:
- `PAPER_TRAIN_ENTRY > 0`
- `PAPER_EXIT > 0`
- `LEARNING_UPDATE ok=True > 0`
- `LEARNING_UPDATE_ERROR = 0`
- `BUCKET_METRICS_ERROR = 0`
- `A_STRICT_TAKE = 0`

New issue:
- Logs are flooded by:
  `[LIVE_ORDER_DISABLED] symbol=... mode check failed (TRADING_MODE required, real orders disabled)`
- This means accepted signals in `paper_train` still enter the live execution/order branch.
- Safety guard blocks real orders, but routing is wrong and noisy.

## Hard rules
- Do not enable real orders.
- Do not change strategy thresholds.
- Do not add new buckets unless already existing.
- Do not modify learning math.
- Do not increase Firebase writes.
- No broad refactor. Surgical mode-routing fix only.

## Files likely involved
- `src/services/trade_executor.py`
- maybe `src/services/runtime_mode.py`
- maybe `src/services/paper_trade_executor.py`

## Required fix

### 1. Centralize mode decision
At the start of `handle_signal()` or before any order placement branch, compute:

```python
mode = get_trading_mode()
paper_train = mode == "paper_train"
paper_live = mode == "paper_live"
live_real = mode == "live_real"
paper_mode = paper_train or paper_live
```

Use existing helpers if available. Do not duplicate env parsing if `runtime_mode.py` already provides this.

### 2. For paper_train ACCEPT/TAKE
When RDE returns ACCEPT/TAKE and `paper_train=True`:

- Open a paper position using existing `open_paper_position()` or existing training router.
- Mark metadata:
  - `paper_source="training_sampler"` or existing canonical paper source
  - `training_bucket="A_STRICT_TAKE"` only if this is truly accepted TAKE
  - `original_decision="TAKE"`
  - `reject_reason=None`
  - `side_inferred=False` unless actually inferred
  - preserve symbol, side, regime, features, score, EV, expected_move if available
- Log:
  `[PAPER_TRAIN_TAKE_ENTRY] bucket=A_STRICT_TAKE symbol=... side=... price=...`
- Then `return`.
- It must not continue into live order code.

### 3. For paper_live ACCEPT/TAKE
If `paper_live=True`, accepted TAKE should open/track paper trade if current architecture supports it, or cleanly return with a single throttled log. It must not spam live order disabled.

### 4. For live_real only
Only when all are true:
- `TRADING_MODE=live_real`
- `ENABLE_REAL_ORDERS=true`
- `LIVE_TRADING_CONFIRMED=true`

then and only then allow live order placement.

Otherwise return before live order placement.

### 5. Add fallback log throttle
Keep `[LIVE_ORDER_DISABLED]` as a final safety guard, but throttle it:
- max once per symbol per 60 seconds
- include mode and flags
- no per-tick spam

Example:

```text
[LIVE_ORDER_DISABLED] symbol=BTCUSDT mode=paper_train real_orders=False confirmed=False reason=not_live_real
```

### 6. Tests
Add tests for:
1. `paper_train + TAKE` opens paper/training position and does not call live order function.
2. `paper_train + duplicate/reject` still routes through sampler.
3. `paper_live` does not call live order function.
4. `live_real + flags false` does not call live order function.
5. `live_real + all flags true` is the only path that can call live order function.
6. `LIVE_ORDER_DISABLED` log is throttled.

## Success definition
`paper_train` must behave as:

```text
signal → RDE decision → paper entry → paper exit → learning update
```

Never:

```text
signal → RDE ACCEPT → live order attempt → LIVE_ORDER_DISABLED spam
```
