# V10.13u+14 Phase 2 — Partial TP Full-Close Lock Bypass Fix

## Production evidence

Running commit before/around Phase 1:
- `30ae68f`
- Runtime restarts observed after deployment.

Critical log sequence:

```text
[CLOSE_LOCK_STALE_RELEASE] key=DOTUSDT:SELL:1.268007 symbol=DOTUSDT reason=PARTIAL_TP_25 age=21.0s count=1
[CLOSE_LOCK_ACQUIRED] DOTUSDT reason=PARTIAL_TP_25 key=DOTUSDT:SELL:1.268007
[CLOSE_LOGIC_START] DOTUSDT reason=PARTIAL_TP_25 entering close logic
[CLOSE_SKIP_DUPLICATE] DOTUSDT reason=PARTIAL_TP_25 key=DOTUSDT:SELL:1.268007 status=already_closing age=10.5s attempts=17
[CLOSE_LOCK_STALE_RELEASE] key=DOTUSDT:SELL:1.268007 symbol=DOTUSDT reason=PARTIAL_TP_25 age=20.3s count=2
[CLOSE_LOCK_ACQUIRED] DOTUSDT reason=PARTIAL_TP_25 key=DOTUSDT:SELL:1.268007
[CLOSE_LOGIC_START] DOTUSDT reason=PARTIAL_TP_25 entering close logic
[CLOSE_SKIP_DUPLICATE] DOTUSDT reason=TRAIL_PROFIT key=DOTUSDT:SELL:1.268007 status=already_closing age=10.7s attempts=87
```

## Diagnosis

V10.13u+14 Phase 1 goal was: partial TP exits must bypass the full-position close lock.

Production proves this is NOT true:
- `PARTIAL_TP_25` still acquires `CLOSE_LOCK`.
- It uses the same key as full close: `symbol:side:entry`.
- Later `TRAIL_PROFIT` is blocked by the partial-TP lock.
- This causes stale release loops and false stuck-position recovery.
- Therefore the bug is not primarily `TRAIL_PROFIT`; root cause is partial TP entering full close lock path.

## Objective

Fix partial TP semantics surgically:
- Partial TP must never acquire or hold full close lock.
- Partial TP must not call full close flow.
- Partial TP must use its own partial-scale path only.
- Full close types must remain lock-protected.

## Files to inspect/modify

- `src/services/trade_executor.py`
- tests in `tests/test_v10_13u_patches.py`

Do not modify:
- PF formula
- EV-only logic
- Firebase read/write behavior
- strategy scoring
- TP/SL distances
- position sizing
- canonical PnL formulas, except if only needed for partial TP bookkeeping consistency

## Required changes

### 1. Normalize exit reason before lock checks

Create/ensure helper:

```python
def _normalize_exit_type(reason: str) -> str:
    r = str(reason or "").upper()
    if r == "REPLACED":
        return "REPLACED_EXIT"
    return r
```

Define constants near close-lock state:

```python
PARTIAL_CLOSE_TYPES = {"PARTIAL_TP_25", "PARTIAL_TP_50", "PARTIAL_TP_75"}
FULL_CLOSE_TYPES = {
    "TP", "SL", "MICRO_TP", "TRAIL_PROFIT", "SCRATCH_EXIT",
    "STAGNATION_EXIT", "TIMEOUT_LOSS", "TIMEOUT_FLAT", "TIMEOUT_PROFIT",
    "EARLY_STOP", "WALL_EXIT", "REPLACED_EXIT", "FORCE_RECONCILE",
}
```

### 2. Hard bypass partial TP before full close lock

At the earliest possible point in close handler, before:
- `_try_acquire_close_lock`
- `_close_key`
- `[CLOSE_LOGIC_START]`
- `build_exit_ctx`
- full position remove

Add:

```python
_exit_type = _normalize_exit_type(reason)

if _exit_type in PARTIAL_CLOSE_TYPES:
    log.warning(
        "[PARTIAL_TP_BYPASS_FULL_CLOSE] %s reason=%s path=partial_only",
        sym, _exit_type
    )
    return _handle_partial_tp_only(sym=sym, pos=pos, curr=curr, reason=_exit_type, ...)
```

Use existing partial TP code. Do not create a second implementation if partial TP logic already exists. Extract existing partial TP logic into `_handle_partial_tp_only()` if needed.

### 3. Partial TP must not remove position

Partial TP may:
- reduce `pos["size"]`
- increment `pos["realized_pnl"]`
- update trailing/entry metadata if already done today
- persist/update metrics if existing logic does so

Partial TP must NOT:
- delete `sym` from `_positions`
- acquire full close lock
- mark full `recently_closed`
- call `update_exit_attribution()` as final full exit
- create final close trade as if whole position closed

If a partial event is recorded, it must be explicitly marked:
- `exit_type="PARTIAL_TP_25"`
- `is_partial=True`
- `final_exit=False`

### 4. Full close lock must reject partial reason defensively

Inside `_try_acquire_close_lock()` or just before calling it, add a hard guard:

```python
if _normalize_exit_type(reason) in PARTIAL_CLOSE_TYPES:
    log.error("[PARTIAL_TP_LOCK_BLOCKED] symbol=%s reason=%s", symbol, reason)
    return False, None, "partial_tp_not_allowed"
```

This prevents future regressions.

### 5. Validate full close types

For full close reasons:
- Continue using close lock.
- Continue try/finally release.
- Keep stale recovery.
- Keep force reconcile only for true full-close stuck states.

### 6. Tests

Add tests:

1. `test_partial_tp25_does_not_acquire_full_close_lock`
   - call close/exit path with `PARTIAL_TP_25`
   - assert `_CLOSING_POSITIONS` unchanged
   - assert no recently_closed full key

2. `test_partial_tp_does_not_remove_position`
   - position remains in `_positions`
   - size decreases or partial handler called

3. `test_trail_profit_can_acquire_after_partial_tp`
   - run partial TP
   - then run `TRAIL_PROFIT`
   - assert full lock can acquire

4. `test_try_acquire_close_lock_rejects_partial_reason`
   - direct guard test

5. `test_replaced_normalizes_to_replaced_exit`
   - regression

6. `test_full_close_still_uses_lock`
   - `SCRATCH_EXIT` or `TRAIL_PROFIT` acquires lock
