# P1.1Z2 — Fix paper state startup order + remove import-time undefined calls

## Problem

Production still logs:

```text
[PAPER_STATE_LOAD_ERROR] source=data/paper_open_positions.json err=name '_reconcile_stale_paper_positions' is not defined
```

Commit: `1f4cf51`

Root cause:
`paper_trade_executor.py` calls `_load_paper_state()` during module import before `_reconcile_stale_paper_positions()` is defined. Previous hotfix moved `_safe_float/_safe_int`, but the same startup-order bug remains for stale reconciliation.

Impact:
- Startup normalization/reconciliation does not run.
- Legacy paper positions with `timeout_s=900` can remain open.
- `C_WEAK_EV_TRAIN` slots get blocked by stale/open positions.
- Bot runs, but paper_train loop is blocked or delayed.

## Hard rules

- Do NOT change strategy logic.
- Do NOT change EV/RDE thresholds.
- Do NOT touch live order behavior except preserving existing safe paper routing.
- Do NOT increase Firebase writes.
- Keep P1.1V learning fix intact.
- Keep P1.1X/P1.1Y routing fixes intact.
- Only fix initialization order, stale reconciliation startup, and paper state safety.

## Required fix

### 1. Remove import-time startup call from the middle of `paper_trade_executor.py`

Find any call like:

```python
_load_paper_state()
```

or similar startup execution that happens before all helper functions are defined.

Move it to the very bottom of the module, after ALL functions are defined, including:

- `_safe_float`
- `_safe_int`
- `_effective_paper_hold_s`
- `_is_position_stale`
- `_normalize_loaded_paper_position`
- `_reconcile_stale_paper_positions`
- `_save_paper_state`
- `close_paper_position`
- any cap-check helpers

### 2. Prefer explicit guarded startup function

Create one startup initializer near the bottom:

```python
_PAPER_STATE_INITIALIZED = False

def _init_paper_state_once() -> None:
    global _PAPER_STATE_INITIALIZED
    if _PAPER_STATE_INITIALIZED:
        return
    _PAPER_STATE_INITIALIZED = True
    try:
        _load_paper_state()
    except Exception as e:
        log.exception("[PAPER_STATE_LOAD_ERROR] source=%s err=%s", PAPER_STATE_FILE, e)
```

Then call it once at the bottom of the file:

```python
_init_paper_state_once()
```

Important: this call must be after all function definitions.

### 3. `_load_paper_state()` must never rely on not-yet-defined names

Inside `_load_paper_state()` it may call:

```python
_normalize_loaded_paper_position(...)
_reconcile_stale_paper_positions(...)
_save_paper_state(...)
```

That is OK only if `_load_paper_state()` is not executed until module bottom.

Do not call `_load_paper_state()` before these functions exist.

### 4. Make stale reconciliation failure non-fatal

Inside `_load_paper_state()`, wrap reconciliation separately:

```python
try:
    summary = _reconcile_stale_paper_positions(source="startup")
    log.info("[PAPER_STATE_RECONCILE_SUMMARY] closed=%d pending=%d alive=%d",
             summary.get("closed", 0), summary.get("pending", 0), summary.get("alive", 0))
except Exception as e:
    log.exception("[PAPER_STATE_RECONCILE_ERROR] err=%s", e)
```

This prevents one reconcile bug from breaking state load.

### 5. Add regression test

Add a test that imports `src.services.paper_trade_executor` fresh and confirms no startup `NameError`:

```python
def test_paper_state_startup_order_no_undefined_reconcile(monkeypatch, tmp_path):
    # simulate legacy paper_open_positions.json with training position timeout_s=900 max_hold_s=300
    # import/reload paper_trade_executor
    # assert no PAPER_STATE_LOAD_ERROR caused by undefined _reconcile_stale_paper_positions
```

Also test:

- `_reconcile_stale_paper_positions` exists before `_init_paper_state_once()` is called.
- stale `C_WEAK_EV_TRAIN` position with age > 300 is closed/reconciled.
- fresh `C_WEAK_EV_TRAIN` position age < 300 remains open.
- stale positions are not counted in max_open caps.
